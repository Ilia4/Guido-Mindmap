## Факты

### Общий контекст
MindMap — один из сервисов внутренней платформы **Guido** (hub нескольких инструментов
под единой авторизацией). Оболочка хаба (роутинг, экран логина, шапка) и центральный
**auth-сервис Guido** — внешние по отношению к сервису; auth-сервис реализует эндпоинт
`GET /auth/me`, принимающий Bearer-токен и возвращающий данные пользователя.

### Frontend (React)
- SPA на **React 19 + Vite + Tailwind CSS**, анимации `framer-motion`, иконки `lucide-react`.
- Встроен в общую оболочку Guido как отдельная страница. Сам пользователя **не логинит** —
  берёт готовый JWT из `localStorage` (ключ `guido_access_token`), который туда положила
  оболочка после входа. Все запросы к API идут с заголовком `Authorization: Bearer <token>`.
- Адрес API берёт из переменной окружения `VITE_API_MINDMAP_BASE`.
- Две ключевые части:
  - **Витрина проектов** (`MindmapPageContent`): список проектов, папки, закрепление,
    тема light/dark, CRUD проектов, архивация, управление доступами (шаринг между
    пользователями), лента уведомлений.
  - **Доска** (`ProjectBoardContent`): карточки на canvas (координаты + размеры), связи
    (рёбра) между карточками, drag&drop, чек-листы, дедлайны, прикрепление документов.
- **Уведомления в реальном времени**: хук слушает Server-Sent Events
  (`GET /api/notifications/stream`) — дедлайны и новые доступы приходят без перезагрузки.

### Reverse-proxy (nginx)
- Терминирует TLS, проксирует `/mindmap/*` на backend API (порт 8088), пробрасывает
  `Upgrade`/`Connection` для SSE.

### Backend API (FastAPI)
- **FastAPI + Uvicorn**, порт `8088`, единый модуль `app.py` (~2900 строк).
- **SQLAlchemy 2.0 Core + PyMySQL**. Таблицы создаются автоматически при старте
  (идемпотентные `CREATE TABLE IF NOT EXISTS`, без отдельных миграций).
- Основная БД **MySQL**. Таблицы: `users`, `projects`, `project_folders`, `project_shares`,
  `cards`, `connections`, `checklists`, `deadlines`, `documents`, `user_notifications`,
  `legacy_sync_state`.
- Авторизация делегирована внешнему auth-сервису: на каждый запрос API вызывает
  `GET {AUTH_SERVICE_URL}/auth/me` с Bearer-токеном; по возвращённому user id находит или
  создаёт локального пользователя.
- Опциональная синхронизация со **старой версией** сервиса через отдельную legacy-БД MySQL
  и каталог загруженных файлов.
- Группы эндпоинтов (префикс `/api`): projects, project-folders, projects/{id}/shares,
  projects/{id}/board, projects/{id}/cards, cards/{id}/documents, notifications
  (+ SSE `notifications/stream`).

### Telegram-бот (отдельный процесс)
- **aiogram 3 + APScheduler**, локальное хранилище SQLite.
- Ежедневная синхронизация по расписанию.
- Общается с: MindMap API (HTTP), напрямую с той же MySQL, внешним auth-сервисом и
  **LLM-мостом** (HTTP-сервис доступа к языковой модели) для генерации/обработки текста.

### Внешние зависимости
- **Auth-сервис Guido** — центральная авторизация (`/auth/me`).
- **LLM-мост** — доступ к языковой модели для бота.

Собери всё в единую связную схему по слоям: клиент → фронтенд → reverse-proxy → backend
(API + бот + MySQL) → внешние сервисы (auth, LLM). Конкретные хосты/IP не важны — используй
обобщённые имена компонентов.
